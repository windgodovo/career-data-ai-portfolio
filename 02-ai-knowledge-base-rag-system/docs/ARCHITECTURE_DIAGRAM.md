## 简化理解

- 用户提问，系统检索，模型生成，返回带引用答案。
- 上到下看：从交互层到接口层，再到核心引擎层、数据层、运维层。
- 核心设计思想：把“问答能力”拆成可独立替换的模块，而不是把所有逻辑堆在一个文件里。
