# 指标与监控

## 1. 指标体系

| 指标 | 计算口径 | 用途 |
|---|---|---|
| 漏过率 | 后验确认违规但策略未及时命中的 case / 后验违规 case | 判断策略覆盖不足 |
| 误杀率 | 复审或申诉后确认正常的处置 case / 处置 case | 判断策略过严 |
| 处置量 | 警告、禁言、下播、封禁等动作数量 | 判断策略影响规模 |
| 申诉率 | 发起申诉 case / 被处置 case | 观察用户反弹 |
| 复审推翻率 | 复审撤销 case / 复审 case | 判断误杀和 SOP 问题 |
| 策略命中确认率 | 命中策略且审核确认违规 case / 策略命中 case | 观察策略准确性 |
| 风险规模趋势 | 按日/周/活动期观察风险 case 数 | 发现突增和黑产活动 |
| 人审压力 | 待审 case 数、平均审核时长 | 评估运营承载 |

## 2. 监控分层

### 2.1 必须当天介入

- 某风险类型处置量日环比增长超过 100%。
- 误杀率或复审推翻率突然升高。
- 活动期、热点事件期间涉政/诈骗/色情风险突增。
- 某国家/语种/房间类型出现集中爆发。
- 人审积压超过 SLA。

### 2.2 可以次日优化

- 某条规则命中确认率持续下降。
- 某类变体话术漏过增加。
- 白名单维护不及时导致轻微误伤。
- 某房间类型策略覆盖不足但规模可控。

## 3. 监控看板建议

```text
总览：风险总量、处置量、漏过率、误杀率、申诉率
风险类型：涉政、色情、暴力、诈骗、诱导、黑灰产
场景维度：直播间、语音房、私信、支付、账号注册
主体维度：国家、语种、房间类型、账号年龄、设备簇、IP 簇
策略维度：规则 ID、命中量、确认率、误杀率、复审推翻率
```

## 4. 报警规则示例

| 报警名称 | 条件 | 动作 |
|---|---|---|
| 风险突增报警 | 风险 case 今日量 > 近 7 日均值 × 2 | 通知策略 + 审核负责人 |
| 误杀突增报警 | 复审推翻率 > 阈值 | 暂停放量，回看样本 |
| 黑产团伙报警 | 单设备簇关联风险账号数 > 阈值 | 拉取图谱并加入黑名单候选 |
| 人审积压报警 | 待审 case 超过 SLA | 临时调整队列优先级 |
