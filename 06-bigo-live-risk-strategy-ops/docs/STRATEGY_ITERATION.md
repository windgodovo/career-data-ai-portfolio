# 策略与规则迭代

## 1. 从洞察到策略

策略迭代的核心是：把分析结论转成可执行、可灰度、可监控、可复盘的规则或机制。

```text
数据洞察
  ↓
风险假设
  ↓
规则/阈值/名单/图谱方案
  ↓
离线回放评估
  ↓
灰度上线
  ↓
指标回看
  ↓
放量或回滚
```

## 2. 策略类型

| 类型 | 适用场景 | 示例 |
|---|---|---|
| 阈值规则 | 明确行为强度异常 | 新号 1 小时内私信超过阈值且被举报 |
| 黑名单 | 高置信风险主体 | 高危设备、IP、收付款账号、支付类别、外链域名 |
| 白名单 | 降低误杀 | 历史稳定主播、官方活动账号 |
| 关联图谱 | 团伙识别 | 账号-设备-IP-支付-房间关系 |
| 分层处置 | 风险等级不同 | 警告、禁言、限流、下播、封禁、复审 |
| 人审兜底 | 规则置信度不足 | 中风险进入人工复审队列 |

## 3. 召回与误杀权衡

| 目标 | 可能风险 | 控制方式 |
|---|---|---|
| 提高召回 | 正常用户被打断 | 灰度、小流量、白名单、人工复审 |
| 降低误杀 | 风险漏过增加 | 分层处置，不同风险等级不同动作 |
| 快速处置 | 审核压力增加 | 高置信自动处置，中置信排队复审 |
| 覆盖黑产变体 | 规则膨胀 | 标签体系和案例库统一维护 |

## 4. 灰度上线流程

1. 离线回放：用历史 7-30 天数据评估命中量、误杀风险、覆盖风险类型。
2. 小流量灰度：选择高风险语种、国家或房间类型试点。
3. 指标回看：观察漏过率、误杀率、申诉率、复审推翻率。
4. 调阈值：根据结果调整阈值或增加白名单。
5. 放量：逐步扩到更多国家、语种、房间类型。
6. 复盘：记录策略收益、风险和后续优化。
