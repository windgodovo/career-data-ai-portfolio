# 跨部门协同手册

## 1. 协同对象

| 团队 | 协同重点 |
|---|---|
| 审核运营 | 判定标准、复审队列、误杀反馈、SOP 更新 |
| 安全运营 | 团伙线索、设备/IP/收付款账号/支付类别、黑灰产链路 |
| 产品 | 入口限制、私信限制、开播门槛、用户教育 |
| 直播业务 | 活动期间策略影响、主播体验、正常流量保护 |
| 数据 / 算法 | 特征稳定性、模型召回、策略效果评估 |
| 客服 / 申诉 | 申诉原因、用户反馈、误杀样本回流 |

## 2. 沟通模板

### 2.1 给审核团队

```text
本次风险主要集中在语音房新号色情引流，变体话术较多。
建议审核重点关注：昵称/头像暗示 + 公屏变体联系方式 + 私信引导组合。
中风险进入复审，高风险直接下播或禁言。
```

### 2.2 给产品团队

```text
数据发现注册 24 小时内新号在语音房高频私信的风险率显著高于成熟账号。
建议对新号私信频次增加冷启动限制，并对高风险房间类型做灰度。
预计对正常主播影响较小，但需要观察新用户互动转化。
```

### 2.3 给直播业务团队

```text
策略目标不是压低直播活跃，而是减少高风险引流对用户体验和平台安全的伤害。
我们会通过白名单、灰度范围和复审机制保护稳定主播和活动账号。
```

## 3. 数据结论转业务语言

| 数据说法 | 业务说法 |
|---|---|
| precision 下降 | 策略最近误伤变多，需要复核边界 |
| recall 不足 | 有一批风险没有被及时拦住 |
| 某设备簇异常 | 可能存在团伙批量养号或批量作案 |
| 复审推翻率高 | 审核标准或策略阈值需要调整 |
| 风险 case 环比增长 | 今天需要介入，而不是等周报 |

## 4. 会议节奏

- 每日：高风险突增和误杀报警同步。
- 每周：策略效果复盘和 bad case 汇总。
- 活动前：特殊活动风控预案。
- 活动后：风险规模、策略效果和用户体验复盘。
