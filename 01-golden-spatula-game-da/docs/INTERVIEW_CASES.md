# 面试典型题与回答框架

## 1. 某版本后 XX 阵容前四率暴跌，如何排查？

回答结构：

1. 先确认数据口径：版本窗口、模式、段位、地区、样本量。
2. 拆分指标：出场率、前四率、登顶率、平均名次分别如何变化。
3. 看阵容组成：核心棋子、羁绊、装备是否被补丁影响。
4. 看经济节奏：成型回合、D 牌回合、升级人口是否变化。
5. 看对手环境：克制阵容是否崛起。
6. 看玩家迁移：原阵容玩家是否换到其他阵容。
7. 输出归因和建议：数值削弱、环境变化、装备变化、运营节奏变化。

## 2. 某棋子登顶率过高，是否一定要削？

不一定。需要区分：

- 出场率是否也高。
- 是否只在高分段强。
- 是否依赖特定装备或三星。
- 是否存在明显克制阵容。
- 是否破坏阵容多样性。

如果“高出场 + 高登顶 + 高前四 + 无明显克制”，才更倾向削弱。

## 3. 海外 TFT 和国服金铲铲有什么分析差异？

- 玩家节奏不同：国服可能更快追 meta，阵容集中更明显。
- 版本节奏不同：上线时间、活动、商业化内容可能不同。
- 模式差异：国服活动模式和社交玩法可能更强。
- 付费差异：皮肤、通行证、活动礼包结构不同。
- 需要统一底层口径，同时保留 region / version / mode 维度。

## 4. 怎么设计制作人看的 Tableau 看板？

制作人看板要少而准：

- 第一屏：DAU、收入、对局数、留存、异常告警。
- 第二屏：版本前后核心指标变化。
- 第三屏：阵容 Top 榜和风险阵容。
- 第四屏：策划行动建议和观察清单。

重点是给决策，不是堆图。
