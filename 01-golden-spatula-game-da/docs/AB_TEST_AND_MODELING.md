# A/B 测试与基础建模

## 1. A/B 测试场景

| 场景 | 实验目标 | 指标 |
|---|---|---|
| 新手引导优化 | 提升新手留存 | 次留、7 留、首局完成率 |
| 商城礼包调整 | 提升付费转化 | 付费率、ARPU、购买转化 |
| 活动入口调整 | 提升活动参与 | 点击率、参与率、奖励领取率 |
| 平衡调整灰度 | 降低异常阵容强度 | 出场率、前四率、登顶率 |

## 2. A/B 评估步骤

1. 确认随机分组是否均衡。
2. 排除版本、平台、地区、段位等混杂因素。
3. 计算核心指标差异。
4. 做显著性检验或置信区间。
5. 判断业务收益和风险。
6. 输出是否放量、继续观察或回滚。

## 3. 流失预警特征

- 最近 7 天登录天数。
- 最近 7 天对局数。
- 最近一次登录距今天数。
- 连败次数或连续低名次次数。
- 付费后活跃变化。
- 新赛季是否回流。

## 4. LTV 预测特征

- 首日对局数。
- 首日游戏时长。
- 首周登录天数。
- 是否购买首充 / 通行证。
- 段位成长速度。
- 活动参与频次。

## 5. 简单模型建议

| 任务 | 模型 |
|---|---|
| 流失预警 | Logistic Regression / Random Forest |
| LTV 预测 | Linear Regression / LightGBM |
| 玩家分群 | KMeans / RFM 分层 |
| 阵容聚类 | 基于棋子/羁绊向量聚类 |
