# 金铲铲项目组 DA：版本分析与游戏数据体系项目

这是一个面向《金铲铲之战》/ 自走棋类游戏的数据分析（DA）作品集项目。项目模拟金铲铲项目组 DA 的核心工作：负责国内版 + 海外版数据体系建设、版本/节点分析、日常监测和策划决策支持，目标是用数据帮助策划判断“改什么、怎么改、改完有没有效果”。

项目重点不是泛泛做游戏报表，而是围绕自走棋业务理解展开：**卡池概率、经济运营、羁绊联动、阵容 meta 迁移、棋子强度平衡、装备组合变化、版本补丁影响**。

## 1. 岗位场景

金铲铲项目组 DA 通常服务这些对象：

- 系统策划：关注经济系统、卡池概率、升级节奏、对局时长。
- 战斗 / 数值策划：关注棋子强度、羁绊强度、装备组合、阵容平衡。
- 版本策划：关注赛季更新、补丁调整、新机制上线后的数据反馈。
- 运营团队：关注活跃、付费、活动参与、回流、流失。
- 制作人 / 项目负责人：关注 DAU、留存、收入、版本健康度和风险点。

## 2. 核心工作内容

### 2.1 打点与看板

- 设计对局、阵容、棋子、羁绊、装备、经济、付费、活动等埋点。
- 建 DA 页面，沉淀版本分析、日常监测和专题分析入口。
- 搭 Tableau / Excel 看板，给制作人和策划快速查看核心指标。

### 2.2 日常监测

每日监控：

- 活跃：DAU、留存、回流、流失。
- 付费：付费人数、付费率、ARPU、ARPPU、礼包转化。
- 对局：开局数、完局率、平均时长、段位分布、模式分布。
- 游戏健康度：阵容集中度、强势阵容占比、异常胜率、补丁后波动。

### 2.3 版本分析

每个赛季 / 补丁更新后快速输出：

- 阵容胜率、前四率、登顶率、出场率变化。
- 羁绊强度、羁绊搭配、羁绊激活阶段表现。
- 棋子强度、装备组合、三星率、登场阶段。
- 经济系统影响：升级节奏、D 牌频率、金币留存、平均人口节奏。
- meta 迁移：玩家从哪些阵容迁移到哪些阵容。

### 2.4 决策支撑

用数据告诉策划：

- 哪个棋子强度过高，需要削弱。
- 哪个羁绊出场率低但胜率高，可能存在理解门槛或冷门强度。
- 哪个阵容前四率暴跌，是棋子削弱、装备变化、卡池概率还是经济节奏导致。
- 哪个活动提升了活跃但没有提升付费，需要调整奖励或入口。

## 5. 核心分析模块

| 模块 | 分析问题 | 输出 |
|---|---|---|
| 活跃与留存 | 新赛季是否带来回流？补丁后是否流失？ | DAU、留存、回流、流失预警 |
| 付费分析 | 新礼包或通行证是否提升付费？ | 付费率、ARPU、LTV、转化漏斗 |
| 对局分析 | 对局时长是否健康？完局率是否下降？ | 对局数、完局率、时长、模式分布 |
| 阵容分析 | 哪些阵容过强或过弱？ | 出场率、前四率、登顶率、胜率 |
| 羁绊分析 | 哪些羁绊组合破坏平衡？ | 激活率、阶段表现、组合胜率 |
| 经济分析 | 升级/D 牌节奏是否变化？ | 平均人口、金币留存、D 牌频率 |
| 版本分析 | 补丁影响了哪些玩法？ | 版本前后对比、meta 迁移报告 |
| A/B 测试 | 调整是否有效？ | 实验组/对照组差异和显著性结论 |

## 6. 典型面试题：某版本后 XX 阵容前四率暴跌，如何排查？

排查框架：

1. 先确认口径：前四率是否分段位、模式、地区、版本、样本量。
2. 拆分变化：出场率下降还是出场仍高但表现下降。
3. 看阵容组成：核心棋子、关键羁绊、装备组合是否被改动。
4. 看经济节奏：该阵容依赖快 8、D 牌、三星还是连胜经济。
5. 看对手环境：是否有克制阵容崛起，导致该阵容被 meta 针对。
6. 看玩家迁移：原来玩该阵容的人转向了哪些阵容。
7. 输出结论：是数值削弱、环境变化、装备变动、卡池概率还是运营节奏导致。

## 8. 可运行样例数据与 Dashboard

项目新增了一套可直接运行的样例数据，参考常见自走棋数据站的展示口径，包括阵容登顶占比、前四占比、出场率、平均名次、版本趋势、补丁前后变化和玩家迁移。

生成 dashboard 数据：

```bash
cd "01-golden-spatula-game-da"
python3 etl/build_dashboard_data.py
```

生成后会在 `data/output/` 产出：

- `comp_meta_summary.csv`：当前版本阵容 meta 汇总。
- `patch_delta_summary.csv`：补丁前后阵容表现变化。
- `meta_migration_summary.csv`：玩家从旧阵容迁移到新阵容的流向。
- `unit_item_risk.csv`：棋子 + 装备组合的强弱风险。

查看 dashboard：

```bash
python3 -m http.server 8011
```

然后打开：

- `http://localhost:8011/dashboard/meta_dashboard.html`
- `http://localhost:8011/dashboard/patch_impact_dashboard.html`

两个 dashboard 的定位：

| Dashboard | 用途 |
|---|---|
| `meta_dashboard.html` | 看当前版本阵容生态：登顶占比、前四占比、出场率、平均名次和强弱标签 |
| `patch_impact_dashboard.html` | 看补丁效果：前四率变化、登顶率变化、出场量变化、玩家迁移和棋子装备风险 |
